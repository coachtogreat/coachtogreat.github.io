Properti Status Dinamis dalam WAI-ARIA

1. Pengenalan Properti Status Dinamis

Properti status dinamis adalah atribut ARIA yang digunakan untuk menggambarkan kondisi atau state elemen yang dapat berubah selama interaksi pengguna. Properti ini membantu memberikan informasi real-time tentang kondisi elemen kepada pengguna dengan teknologi bantuan.

2. Jenis-jenis Properti Status Dinamis

  • aria-expanded: Menunjukkan apakah elemen dapat diperluas atau dikompres
  • aria-selected: Menunjukkan apakah elemen terpilih
  • aria-checked: Menunjukkan status kotak centang atau radio button
  • aria-pressed: Menunjukkan status tombol yang dapat ditekan
  • aria-disabled: Menunjukkan apakah elemen dinonaktifkan
  • aria-invalid: Menunjukkan apakah ada error pada elemen
  • aria-hidden: Menunjukkan apakah elemen disembunyikan
  • aria-current: Menunjukkan elemen yang sedang aktif

3. Contoh Penggunaan Properti Status Dinamis

Contoh Tab Panel dengan Properti Status 
<div role="tablist" aria-label="Menu Produk">
  <button role="tab" aria-selected="true" aria-controls="panel1">Produk 1</button>
  <button role="tab" aria-selected="false" aria-controls="panel2">Produk 2</button>
</div>

<div role="tabpanel" id="panel1" aria-labelledby="tab1">
  Konten Produk 1
</div>

<div role="tabpanel" id="panel2" aria-labelledby="tab2">
  Konten Produk 2
</div>

4. Properti Status dalam Form Validasi

Contoh Form dengan Properti Status 
<form role="form" aria-label="Form Registrasi">
  <div>
    <label for="email">Email</label>
    <input type="email" id="email" 
           aria-invalid="false" 
           aria-describedby="email-error">
    <span id="email-error" aria-live="polite"></span>
  </div>

  <div>
    <label for="password">Password</label>
    <input type="password" id="password" 
           aria-invalid="false" 
           aria-describedby="password-error">
    <span id="password-error" aria-live="polite"></span>
  </div>
</form>

5. Properti Status dalam Menu

Contoh Menu dengan Properti Status 
<nav role="navigation" aria-label="Menu Navigasi">
  <button aria-expanded="false" aria-controls="menu-items">Menu</button>
  
  <div id="menu-items" role="menu" hidden>
    <button role="menuitem" aria-current="page">Beranda</button>
    <button role="menuitem">Produk</button>
    <button role="menuitem">Kontak</button>
  </div>
</nav>

6. Best Practices Penggunaan Properti Status

  • Gunakan properti status yang sesuai dengan konteks
  • Update status secara dinamis
  • Pastikan status selalu akurat
  • Gunakan aria-live untuk update status
  • Dokumentasikan perubahan status

7. Tips Penggunaan Properti Status

  • Gunakan JavaScript untuk mengelola status
  • Pastikan update status terdeteksi
  • Gunakan event listeners untuk perubahan
  • Test dengan screen reader
  • Dokumentasikan implementasi

8. Dampak Properti Status pada Aksesibilitas

  • Meningkatkan pemahaman kondisi elemen
  • Membantu navigasi yang lebih efisien
  • Membuat interaksi lebih terstruktur
  • Meningkatkan pengalaman pengguna
  • Membantu pengguna dengan teknologi bantuan

9. Contoh Implementasi Properti Status

Contoh Implementasi JavaScript 
// JavaScript untuk mengelola status
function toggleMenu(button, menu) {
    const expanded = button.getAttribute('aria-expanded') === 'true';
    button.setAttribute('aria-expanded', !expanded);
    menu.hidden = expanded;
}

// Form validation
function validateForm() {
    const email = document.getElementById('email');
    const emailError = document.getElementById('email-error');
    
    if (!isValidEmail(email.value)) {
        email.setAttribute('aria-invalid', 'true');
        emailError.textContent = 'Email tidak valid';
    } else {
        email.setAttribute('aria-invalid', 'false');
        emailError.textContent = '';
    }
}

10. Kapan Menggunakan Properti Status

  • Pada elemen yang dapat berubah kondisinya
  • Ketika perlu memberikan feedback
  • Untuk meningkatkan navigasi
  • Pada komponen interaktif
  • Untuk memberikan konteks tambahan
