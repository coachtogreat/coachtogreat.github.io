Konsistensi dalam Penulisan Kode

Panduan Menjaga Konsistensi Kode HTML yang Bersih dan Terstruktur

Konsistensi dalam penulisan kode adalah kunci untuk membuat kode yang mudah dibaca, dipelihara, dan dikembangkan lebih lanjut. Kode yang konsisten membantu tim pengembang untuk berkolaborasi dengan lebih baik dan mengurangi kemungkinan kesalahan.

Mengapa Konsistensi Kode Penting?
  • Memudahkan pemeliharaan kode
  • Meningkatkan keterbacaan kode
  • Mempercepat proses pengembangan tim
  • Mengurangi kemungkinan bug
  • Memudahkan onboarding pengembang baru

1. Konsistensi Struktur Dokumen

Pertahankan struktur dokumen yang konsisten di seluruh proyek.

HTML 
<!DOCTYPE html>
<html lang="id">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Judul Halaman</title>
  <link rel="stylesheet" href="styles.css">
  <!-- CSS tambahan -->
</head>
<body>
  <!-- Header -->
  <header>...</header>
  
  <!-- Konten Utama -->
  <main>
    <article>...</article>
    <aside>...</aside>
  </main>
  
  <!-- Footer -->
  <footer>...</footer>
  
  <!-- JavaScript -->
  <script src="script.js"></script>
</body>
</html>

2. Aturan Penamaan yang Konsisten

Pilih dan patuhi konvensi penamaan yang konsisten.

1. Nama Kelas (Class Names)
✅ Baik: <div class="header-navigation">...</div> <button class="btn btn-primary">Kirim</button> <section class="user-profile">...</section>
❌ Buruk: <div class="headerNav">...</div> <button class="button_primary">Kirim</button> <section class="UserProfile">...</section>
2. ID dan Atribut Data
✅ Baik: <div id="main-navigation">...</div> <div data-user-id="123">...</div> <button data-action="submit-form">Kirim</button>
❌ Buruk: <div id="mainNav">...</div> <div data-userid="123">...</div> <button dataAction="submitForm">Kirim</button>

3. Formatting dan Spasi

Pertahankan konsistensi dalam formatting dan spasi.

HTML 
<!-- Gunakan indentasi 2 spasi -->
<div class="container">
  <header class="page-header">
    <h1>Judul Halaman</h1>
  </header>
  
  <!-- Baris kosong di antara bagian yang berbeda -->
  <main class="content">
    <article>
      <h2>Artikel Terbaru</h2>
      <p>Isi artikel...</p>
    </article>
    
    <aside class="sidebar">
      <h3>Widget</h3>
      <ul>
        <li>Item 1</li>
        <li>Item 2</li>
      </ul>
    </aside>
  </main>
</div>

4. Atribut HTML yang Konsisten

Gunakan atribut dengan cara yang konsisten di seluruh proyek.

1. Atribut Boolean
✅ Baik: <input type="checkbox" checked> <button disabled>Kirim</button> <video controls>...</video>
❌ Buruk: <input type="checkbox" checked="checked"> <button disabled="true">Kirim</button> <video controls="controls">...</video>
2. Kutipan Atribut
✅ Baik: <a href="profile.html" title="Profil Pengguna">...</a> <img src="image.jpg" alt="Deskripsi gambar">
❌ Buruk: <a href='profile.html' title='Profil Pengguna'>...</a> <img src=image.jpg alt=Deskripsi gambar>

5. Organisasi Kode

Kelompokkan dan urutkan bagian-bagian kode dengan konsisten.

HTML 
<!-- 1. DOCTYPE dan HTML -->
<!DOCTYPE html>
<html lang="id">

<!-- 2. Head Section -->
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Judul Halaman</title>
  
  <!-- CSS -->
  <link rel="stylesheet" href="css/reset.css">
  <link rel="stylesheet" href="css/main.css">
  
  <!-- Favicon -->
  <link rel="icon" href="favicon.ico">
</head>

<!-- 3. Body Section -->
<body>
  <!-- Header -->
  <header class="main-header">...</header>
  
  <!-- Navigasi -->
  <nav class="main-nav">...</nav>
  
  <!-- Konten Utama -->
  <main class="main-content">...</main>
  
  <!-- Footer -->
  <footer class="main-footer">...</footer>
  
  <!-- JavaScript -->
  <script src="js/vendor.js"></script>
  <script src="js/main.js"></script>
</body>
</html>

6. Alat untuk Menjaga Konsistensi

Alat Deskripsi Kegunaan
Prettier Formatter kode otomatis Format kode secara konsisten di seluruh proyek
ESLint Linter untuk JavaScript dan JSX Menemukan dan memperbaiki masalah dalam kode
Stylelint Linter untuk CSS/SCSS Memastikan konsistensi gaya CSS
EditorConfig Konfigurasi editor yang konsisten Menjaga konsistensi pengaturan editor di seluruh tim

7. Contoh File .editorconfig

.editorconfig 
# EditorConfig helps developers define and maintain consistent
# coding styles between different editors and IDEs
# editorconfig.org

root = true

[*]
# Change these settings to your own preference
indent_style = space
indent_size = 2

# We recommend you to keep these unchanged
end_of_line = lf
charset = utf-8
trim_trailing_whitespace = true
insert_final_newline = true

[*.{js,jsx,ts,tsx,vue}]
indent_size = 2

[*.{html,htm}]
indent_size = 2

[*.{css,scss,less}]
indent_size = 2

[*.md]
trim_trailing_whitespace = false
Hal yang Harus Dihindari:
  • Mencampur berbagai gaya penulisan kode dalam satu proyek
  • Mengabaikan konvensi tim yang sudah disepakati
  • Tidak mengikuti pedoman gaya yang sudah ditetapkan
  • Mengubah gaya penulisan hanya untuk bagian kode tertentu
  • Tidak menggunakan alat bantu untuk menjaga konsistensi
Tips Menjaga Konsistensi:
  1. Buat dan ikuti pedoman gaya (style guide) tim
  2. Gunakan alat otomatisasi seperti Prettier dan ESLint
  3. Lakukan code review secara teratur
  4. Dokumentasikan keputusan terkait gaya kode
  5. Lakukan refactoring berkala untuk menjaga konsistensi
