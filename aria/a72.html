Studi Kasus dan Contoh Pengujian ARIA

1. Pengenalan Studi Kasus

Studi kasus akan membantu memahami penerapan praktis pengujian ARIA dalam berbagai situasi. Artikel ini akan membahas beberapa studi kasus dan contoh pengujian yang dapat menjadi referensi.

2. Studi Kasus 1: E-commerce Website

Contoh Pengujian Produk Grid 
// Struktur HTML Produk
<div role="grid" aria-label="Daftar Produk">
  <div role="row">
    <div role="columnheader">Gambar</div>
    <div role="columnheader">Nama</div>
    <div role="columnheader">Harga</div>
  </div>
  <div role="row">
    <div role="gridcell">
      <img src="product1.jpg" alt="Produk 1">
    </div>
    <div role="gridcell">Produk 1</div>
    <div role="gridcell">Rp 100.000</div>
  </div>
</div>

3. Studi Kasus 2: Formulir Pendaftaran

Contoh Pengujian Form Validasi 
// Form dengan Validasi
<form role="form" aria-label="Form Pendaftaran">
  <div>
    <label for="email">Email</label>
    <input type="email" id="email" 
           aria-invalid="false" 
           aria-describedby="email-error">
    <span id="email-error" aria-live="polite"></span>
  </div>
</form>

4. Studi Kasus 3: Menu Navigasi

Contoh Pengujian Menu 
// Menu dengan ARIA
<nav role="navigation" aria-label="Menu Navigasi">
  <button aria-expanded="false" aria-controls="menu-items">
    Menu
  </button>
  
  <div id="menu-items" role="menu" hidden>
    <button role="menuitem">Beranda</button>
    <button role="menuitem">Produk</button>
    <button role="menuitem">Kontak</button>
  </div>
</nav>

5. Contoh Pengujian dengan NVDA

  • Memeriksa navigasi dengan tab
  • Menggunakan shortcut NVDA
  • Mengidentifikasi elemen yang tidak dapat diakses
  • Memeriksa pesan error
  • Menggunakan mode browse dan focus

6. Contoh Pengujian dengan Chrome DevTools

Contoh Penggunaan DevTools 
// Membuka Accessibility Panel
1. Buka Chrome DevTools
2. Klik tab "Accessibility"
3. Pilih elemen yang ingin diuji
4. Lihat report aksesibilitas

// Menggunakan Console
console.log($0.accessibilityNode);
console.log($0.ariaProperties);

7. Contoh Pengujian dengan axe-core

Contoh Penggunaan axe-core 
// Pengujian dengan axe-core
const { axe } = require('@axe-core/puppeteer');

async function testAccessibility() {
  const browser = await puppeteer.launch();
  const page = await browser.newPage();
  await page.goto('https://example.com');
  
  const results = await axe(page);
  console.log(results.violations);
  
  await browser.close();
}

8. Best Practices Pengujian

  • Test dengan berbagai screen reader
  • Gunakan kombinasi alat pengujian
  • Test di berbagai device
  • Dokumentasikan hasil pengujian
  • Buat rencana perbaikan

9. Tips Pengujian Efektif

  • Mulai dengan pengujian manual
  • Gunakan alat otomatis untuk validasi
  • Test dengan pengguna nyata
  • Simulasikan berbagai kondisi
  • Update test cases secara berkala

10. Kesimpulan

Studi kasus dan contoh pengujian yang telah dibahas menunjukkan pentingnya pendekatan terstruktur dalam pengujian ARIA. Dengan menggabungkan pengujian manual dan otomatis, serta menggunakan berbagai alat pengujian, kita dapat memastikan implementasi ARIA yang benar dan meningkatkan aksesibilitas situs web.
