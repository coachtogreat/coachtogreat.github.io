Konvensi Penamaan CSS: BEM dan OOCSS

Konvensi penamaan yang konsisten dalam CSS sangat penting untuk memelihara kode yang bersih, dapat diprediksi, dan mudah dipahami, terutama dalam proyek tim yang besar. Dua pendekatan populer yang akan kita bahas adalah BEM (Block Element Modifier) dan OOCSS (Object-Oriented CSS).

Mengapa Membutuhkan Konvensi Penamaan?

  • Meningkatkan keterbacaan kode
  • Memudahkan kolaborasi tim
  • Mengurangi konflik CSS
  • Mempermudah pemeliharaan jangka panjang
  • Membantu menghindari spesifisitas yang tidak perlu
1. BEM (Block Element Modifier)

BEM adalah metodologi penamaan yang membagi antarmuka pengguna menjadi blok-blok yang independen, membuat kode CSS lebih mudah dipahami dan dikembangkan.

Konsep Dasar BEM:

  • Block: Komponen yang berdiri sendiri dan dapat digunakan kembali (contoh: .card, .header)
  • Element: Bagian dari blok yang tidak memiliki makna tersendiri di luar bloknya (contoh: .card__title, .card__body)
  • Modifier: Variasi atau status dari blok/elemen (contoh: .card--large, .button--disabled)
Contoh Struktur HTML:
<div class="card card--featured">
  <div class="card__header">
    <h3 class="card__title">Judul Kartu</h3>
  </div>
  <div class="card__body">
    <p class="card__text">Ini adalah contoh kartu.</p>
  </div>
  <div class="card__footer">
    <button class="card__button card__button--primary">Tombol</button>
  </div>
</div>
Contoh CSS:
/* Block */
.card {
  border: 1px solid #ddd;
  border-radius: 4px;
  padding: 15px;
}

/* Element */
.card__title {
  margin: 0 0 10px 0;
  font-size: 1.5em;
}

.card__button {
  padding: 8px 16px;
  border: none;
  border-radius: 4px;
}

/* Modifier */
.card--featured {
  border-color: #3498db;
  background-color: #f0f8ff;
}

.card__button--primary {
  background-color: #3498db;
  color: white;
}

Aturan Penamaan BEM:

  • Gunakan huruf kecil dan tanda hubung untuk nama yang terdiri dari beberapa kata
  • Gunakan dua garis bawah (__) untuk memisahkan blok dan elemen
  • Gunakan dua tanda hubung (--) untuk memisahkan elemen dan modifier
  • Hindari nested selector yang dalam
  • Setiap blok harus independen dan dapat digunakan kembali
2. OOCSS (Object-Oriented CSS)

OOCSS adalah metodologi yang memisahkan struktur dan skin (tampilan) untuk menciptakan komponen yang dapat digunakan kembali dan mengurangi duplikasi kode.

Dua Prinsip Utama OOCSS:

  1. Pemisahan Struktur dan Skin:
    • Struktur: Properti layout seperti width, height, margin, padding
    • Skin: Properti tampilan seperti colors, fonts, shadows
  2. Pemisahan Container dan Konten:
    • Hindari gaya yang bergantung pada lokasi elemen
    • Buat komponen yang dapat digunakan di mana saja
Contoh HTML:
<div class="media">
  <img src="image.jpg" class="media__image" alt="">
  <div class="media__content">
    <h3 class="title">Judul Konten</h3>
    <p class="text">Ini adalah contoh konten media.</p>
  </div>
</div>

<div class="media media--reversed">
  <img src="image.jpg" class="media__image" alt="">
  <div class="media__content">
    <h3 class="title">Konten Terbalik</h3>
    <p class="text">Ini adalah contoh konten media yang terbalik.</p>
  </div>
</div>
Contoh CSS:
/* Struktur */
.media {
  display: flex;
  align-items: flex-start;
  margin-bottom: 1em;
}

.media__image {
  margin-right: 1em;
  width: 100px;
  height: 100px;
  object-fit: cover;
}

.media__content {
  flex: 1;
}

/* Skin */
.title {
  color: #333;
  font-size: 1.2em;
  margin: 0 0 0.5em 0;
}

.text {
  color: #666;
  margin: 0;
}

/* Modifier */
.media--reversed {
  flex-direction: row-reverse;
}

.media--reversed .media__image {
  margin-right: 0;
  margin-left: 1em;
}

Perbandingan BEM dan OOCSS

Kelebihan BEM

  • Penamaan yang sangat deskriptif
  • Mudah dipahami oleh pengembang baru
  • Mengurangi kemungkinan konflik CSS
  • Dokumentasi yang baik
  • Cocok untuk tim besar

Kelebihan OOCSS

  • Kode yang lebih ringkas
  • Rasio CSS/HTML yang lebih baik
  • Performa yang lebih baik (ukuran file lebih kecil)
  • Fleksibilitas tinggi
  • Cocok untuk proyek dengan banyak komponen yang dapat digunakan kembali

Praktik Buruk yang Harus Dihindari:

  • Nama kelas yang tidak deskriptif: .box1, .red, .left
  • Selector yang terlalu spesifik: div#header ul.nav li a
  • !important yang berlebihan
  • Menggabungkan struktur dan skin: .sidebar-widget (seharusnya dipisah menjadi .sidebar .widget)

Tips Praktis:

  • Pilih satu metodologi dan konsisten menggunakannya
  • Buat dokumentasi untuk tim Anda
  • Gunakan preprocessor untuk memudahkan penulisan BEM/OOCSS
  • Pertimbangkan untuk menggunakan pendekatan hybrid jika diperlukan
  • Lakukan code review untuk memastikan konsistensi
