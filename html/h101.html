Aksesibilitas Dasar (Alt Text, Label)

Membuat Web yang Dapat Diakses Semua Orang

Aksesibilitas web memastikan bahwa semua orang, termasuk penyandang disabilitas, dapat mengakses dan berinteraksi dengan konten web. Mari pelajari dasar-dasar aksesibilitas dalam HTML.

Catatan: Aksesibilitas bukan hanya untuk pengguna dengan disabilitas, tetapi juga bermanfaat untuk semua pengguna dalam berbagai situasi.

1. Teks Alternatif (Alt Text) untuk Gambar

Atribut alt memberikan deskripsi teks untuk gambar:

HTML 
<!-- Gambar informatif -->
<img 
  src="gambar/logo.png" 
  alt="Logo WebID Design - Desain Web Profesional"
  width="200"
  height="100">

<!-- Gamban dekoratif -->
<img 
  src="gambar/ornamen.png" 
  alt=""
  role="presentation"
  width="100"
  height="50">
Tip: Gunakan alt text yang deskriptif untuk gambar informatif, dan biarkan kosong (alt="") untuk gambar dekoratif.

2. Label untuk Formulir

Setiap elemen input harus memiliki label yang terkait:

HTML 
<!-- Menggunakan for dan id -->
<label for="nama">Nama Lengkap:</label>
<input type="text" id="nama" name="nama">

<!-- Atau membungkus input -->
<label>
  Email:
  <input type="email" name="email" required>
</label>

3. Judul dan Struktur yang Jelas

Gunakan heading dengan urutan yang logis dan deskriptif:

HTML 
<h1>Judul Halaman</h1>
<h2>Bagian Utama</h2>
<h3>Sub-bagian</h3>

<!-- Skip to main content link untuk keyboard users -->
<a href="#main" class="sr-only">Lanjutkan ke konten utama</a>
<nav>...</nav>
<main id="main">...</main>

4. ARIA (Accessible Rich Internet Applications)

Tambahkan semantik tambahan untuk komponen interaktif:

HTML 
<!-- Tab panel dengan ARIA -->
<div role="tablist">
  <button 
    role="tab" 
    aria-selected="true"
    aria-controls="panel1"
    id="tab1">
    Tab 1
  </button>
  <div 
    role="tabpanel" 
    id="panel1" 
    aria-labelledby="tab1"
    tabindex="0">
    Konten tab 1
  </div>
</div>
Penting: Gunakan elemen HTML semantik yang sesuai sebelum beralih ke ARIA. ARIA seharusnya hanya digunakan ketika elemen HTML standar tidak mencukupi.

5. Kontras Warna yang Cukup

Pastikan rasio kontras antara teks dan latar belakang memenuhi standar:

CSS 
/* Kontras yang baik (minimal 4.5:1 untuk teks normal) */
.baik {
  color: #000000;
  background-color: #ffffff; /* Rasio 21:1 */
}

/* Kontras buruk (hanya 2.14:1) */
.buruk {
  color: #777777;
  background-color: #ffffff;
}

6. Fokus Keyboard yang Jelas

Pastikan semua elemen interaktif dapat diakses dengan keyboard:

CSS 
/* Gaya fokus yang terlihat */
:focus {
  outline: 3px solid #4d90fe;
  outline-offset: 2px;
}

/* Jangan pernah menghilangkan outline sepenuhnya! */
*:focus {
  outline: none; /* JANGAN LAKUKAN INI! */
}
Tip: Uji navigasi keyboard di situs Anda dengan menekan Tab. Pastikan semua elemen interaktif dapat diakses dan fokus terlihat jelas.

7. Bahasa dan Baca Arah

Tentukan bahasa halaman dan bagian yang menggunakan bahasa berbeda:

HTML 
<!DOCTYPE html>
<html lang="id">
<head>
  <meta charset="UTF-8">
  <title>Halaman dalam Bahasa Indonesia</title>
</head>
<body>
  <p>Ini adalah konten dalam Bahasa Indonesia.</p>
  
  <!-- Konten dalam bahasa lain -->
  <div lang="en">
    <p>This content is in English.</p>
  </div>
  
  <!-- Teks dengan arah baca kanan ke kiri -->
  <p dir="rtl" lang="ar">هذا النص باللغة العربية</p>
</body>
</html>
Alat Bantu: Gunakan alat seperti WAVE, axe, atau Lighthouse untuk menguji aksesibilitas situs Anda.
