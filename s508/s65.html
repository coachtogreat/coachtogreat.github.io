Penggunaan ARIA yang Tepat dan Valid

1. Pengertian ARIA

ARIA (Accessible Rich Internet Applications) adalah spesifikasi yang membantu membuat aplikasi web lebih aksesibel untuk pengguna dengan keterbatasan. ARIA menambahkan atribut ke HTML untuk membantu screen reader dan alat bantu lainnya.

2. Jenis-jenis ARIA

2.1. Roles

  • Landmark roles (banner, navigation, main)
  • Widget roles (button, checkbox, menu)
  • Document structure roles (article, section, heading)
  • Window roles (dialog, alertdialog, tooltip)

2.2. Properties

  • aria-label
  • aria-labelledby
  • aria-describedby
  • aria-expanded
  • aria-checked
  • aria-selected

2.3. States

  • aria-hidden
  • aria-disabled
  • aria-invalid
  • aria-required
  • aria-selected
  • aria-expanded

3. Best Practices ARIA

3.1. Penggunaan

  • Menggunakan HTML semantik terlebih dahulu
  • Menggunakan ARIA hanya jika diperlukan
  • Menghindari konflik dengan HTML semantik
  • Menggunakan ARIA yang sesuai

3.2. Validasi

  • Menggunakan roles yang valid
  • Menggunakan properties yang sesuai
  • Menggunakan states yang tepat
  • Menghindari penggunaan yang berlebihan

3.3. Testing

  • Menguji dengan screen reader
  • Menguji dengan keyboard
  • Menguji dengan tools aksesibilitas
  • Menguji dengan user testing
